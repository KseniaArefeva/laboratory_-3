# Лабораторная работа №3

## Очередь через два стека (CustomQueue)

### Постановка задачи

Реализовать очередь (FIFO) с помощью двух стеков (LIFO).  
Обязательное требование: добавить проверку на переполнение – при попытке добавить элемент в полностью заполненную очередь должен удаляться самый старый элемент, а новый элемент добавляется в конец. Такой механизм называется перезаписью последнего элемента.

### Пояснения к коду

В предоставленном коде создан класс `CustomQueue`, который внутри использует два списка Python:

- `input_stack` – стек для добавления новых элементов (при вызове `push`).
- `output_stack` – стек для извлечения элементов (при вызове `pop` или `peek`).

### Основные методы

| Метод       | Действие                                                                                     |
|-------------|----------------------------------------------------------------------------------------------|
| `push(x)`   | Добавляет элемент `x` в конец очереди. Если задан `limit` и очередь уже полна – предварительно удаляет самый старый элемент через `pop()`. |
| `pop()`     | Удаляет и возвращает первый элемент очереди. Если очередь пуста – возвращает `None`. |
| `peek()`    | Возвращает первый элемент без удаления. Если очередь пуста – возвращает `None`.              |

### Ключевой принцип работы двух стеков

Очередь имитируется за счёт **ленивого перекладывания** элементов из `input_stack` в `output_stack`:

- При `push` элемент всегда кладётся в `input_stack` (операция O(1)).
- При `pop` или `peek`:
  - Если `output_stack` не пуст – верхний элемент этого стека является головой очереди (берётся или просматривается за O(1)).
  - Если `output_stack` пуст – все элементы из `input_stack` перекладываются в `output_stack` (при этом порядок элементов меняется на противоположный, что восстанавливает FIFO). После этого голова оказывается на вершине `output_stack`.

Такая схема даёт амортизированное время O(1) на каждую операцию.

### Реализация перезаписи при переполнении

В конструкторе задаётся необязательный параметр `limit` (по умолчанию `None` – нет ограничения).  
В методе `push` перед добавлением нового элемента выполняется проверка:

если limit указан и текущий размер очереди == limit:
вызвать pop() # удалить голову.

Таким образом, при попытке добавить элемент в уже заполненную очередь сначала удаляется самый старый элемент, а затем новый добавляется в хвост. Размер очереди остаётся равным `limit`, а старый элемент «перезаписывается» новым.

### Проведённые тесты

Тесты выполнены на очереди с максимальным размером 3.

```python
q = CustomQueue(3)  # очередь вместимостью 3 элемента
q.push('A')
q.push('B')
q.push('C')
print(q.peek())     # ожидаем 'A' – первый добавленный
q.push('D')         # переполнение, удаляется 'A', добавляется 'D'
print(q.pop())      # ожидаем 'B'
print(q.pop())      # ожидаем 'C'
print(q.pop())      # ожидаем 'D'
