# Лабораторная работа №3

## Реализация очереди через два стека (CustomQueue)

### Постановка задачи

Реализовать очередь с дисциплиной обслуживания FIFO (первым пришёл – первым ушёл) с помощью двух стеков LIFO (последним пришёл – первым ушёл).  

**Дополнительное требование:** добавить механизм контроля переполнения – если очередь имеет ограниченный максимальный размер и при добавлении нового элемента этот размер достигнут, то самый старый элемент должен автоматически удаляться, а новый элемент добавляться в конец очереди (механизм перезаписи).

### Обоснование выбора структуры данных

Два стека выбраны потому, что:
- Стек сам по себе даёт только LIFO, но с помощью двух стеков можно эмулировать FIFO.
- Операции со стеком имеют сложность O(1), что обеспечивает высокую производительность.
- Предложенная реализация с "ленивым" перекладыванием элементов даёт амортизированную сложность O(1) для всех операций.

### Описание внутренней структуры

В моём классе `CustomQueue` используются два списка Python:

- **`input_stack`** – стек для добавления новых элементов (при вызове `push`).
- **`output_stack`** – стек для извлечения элементов (при вызове `pop` или `peek`).

Дополнительно в конструкторе передаётся параметр `capacity` (максимальный размер очереди). Если `capacity = None`, ограничения на размер нет.

### Основные методы класса

| Метод        | Назначение                                                                                  |
|--------------|---------------------------------------------------------------------------------------------|
| `push(value)`| Добавляет элемент `value` в конец очереди. Если очередь ограничена и достигнут лимит – предварительно удаляет самый старый элемент через `pop()`. |
| `pop()`      | Удаляет и возвращает первый элемент очереди. Если очередь пуста – возвращает `None`.        |
| `peek()`     | Возвращает первый элемент очереди без его удаления. Если очередь пуста – возвращает `None`. |
| `_current_size()` | Вспомогательный метод, возвращающий текущий размер очереди (сумма длин двух стеков).   |

### Алгоритм работы двух стеков

Ключевая идея: **перенос элементов происходит только при необходимости**.

1. При вызове `push(value)` новый элемент всегда помещается в `input_stack`. Это операция O(1).
2. При вызове `pop()` или `peek()`:
   - Если `output_stack` не пуст – его верхний элемент является головой очереди (для `peek` просто смотрим, для `pop` – удаляем).
   - Если `output_stack` пуст – все элементы из `input_stack` поочерёдно переносятся в `output_stack`. При этом порядок элементов меняется на обратный, что превращает LIFO в FIFO. После этого голова оказывается на вершине `output_stack`.
3. Такой подход даёт амортизированное время O(1) на каждую операцию, потому что каждый элемент переносится из `input_stack` в `output_stack` не более одного раза за всё время жизни очереди.

### Реализация перезаписи при переполнении

В конструкторе задаётся параметр `capacity` (по умолчанию `None` – без ограничений).
