## TwoStackQueue - Очередь на двух стеках с перезаписью

### Постановка задачи

Реализовать очередь FIFO (First-In, First-Out) с помощью двух стеков LIFO (Last-In, First-Out).  
Особенность реализации: при переполнении очереди самый старый элемент автоматически удаляется, а новый добавляется в конец (механизм перезаписи).

### Принцип работы

Класс `TwoStackQueue` использует два списка:
- `stack_add` - стек для добавления новых элементов
- `stack_remove` - стек для извлечения элементов

Элементы перекладываются из одного стека в другой только при необходимости, что обеспечивает амортизированную сложность O(1) для всех операций.

### Методы класса

| Метод | Описание |
|-------|----------|
| `enqueue(value)` | Добавляет элемент в конец очереди. При переполнении удаляет самый старый элемент |
| `dequeue()` | Удаляет и возвращает первый элемент очереди |
| `front()` | Возвращает первый элемент без удаления |
| `is_empty()` | Проверяет, пуста ли очередь |
| `size()` | Возвращает количество элементов |
| `clear()` | Очищает очередь |
| `display()` | Выводит все элементы в порядке от головы к хвосту |
| `contains(value)` | Проверяет наличие элемента в очереди |
| `get_operations_count()` | Возвращает общее количество операций |

### Алгоритм перезаписи при переполнении

```python
def enqueue(self, value):
    if self.capacity is not None and self.size() >= self.capacity:
        self.dequeue()  # удаляем самый старый элемент
    self.stack_add.append(value)
```
### Пример работы 

```python
queue = TwoStackQueue(limit=3)

queue.enqueue('A')
queue.enqueue('B')
queue.enqueue('C')
print(queue.front())  # A

queue.enqueue('D')    # A удаляется, добавляется D
print(queue.size())   # 3

queue.display()       # B -> C -> D

print(queue.dequeue())  # B
print(queue.dequeue())  # C
print(queue.dequeue())  # D

print(queue.is_empty())  # True
print(queue.get_operations_count())  # 7
```
